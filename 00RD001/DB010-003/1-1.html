さまざまなRDBMS

Oracle

非常に高機能なDBMSである。標準SQLとは互換性のない独自構文の利用が求められることもある。
企業に使われるシステムのため、さまざまな要求を満たす拡張機能がある。 現行バージョン(Oracle Database 12c)における特徴としては、行レベルロック、読み取り一貫性、堅牢性、移植性などがある。

Microsoft SQL Server

マイクロソフト系システム開発で幅広く利用されている。関数や型の種類、ロック機構、データベースの自動圧縮機能などの独自性がいくつか存在する。
MicrosoftWindowsとの相性がいい。

PostgreSQL

MySQLと双璧をなすオープンソースのRDBMS製品である。オープンソース製品の中では機能が豊富で標準SQLへの準拠度が高い。
PostGISというGIS(地理情報システム）を扱うアドオンソフトがあり、使い勝手が良い。ほかの特徴としては、サーバで実行される処理のまとまりを定義できる関数などがある。

SQLite

アプリケーションの一部に組み込まれて動作するタイプのオープンソースRDBMSである。
大規模な利用には向かず、多くのDBMSがサポートする機能や関数の中でSQLiteがサポートしていないものがある。その一方で、手軽さと高速な動作から中規模な開発では活用されている。
